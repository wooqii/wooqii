# ‘북클럽' 형식에 관한 프로토콜 (ver.1)


  1. 호칭: ‘(이름)님', ‘존칭'
  2. 행정
    1. 인원:
    2. 참여인원 사전조사
      1. 북클럽 일자 2일 전까지 마감
    3. 최대인원
      1. 선착순
      2. 참여인 (최대 15인)/ 참관인 (제한 없음)
      3. 세션이 변경되면 새로운 정원수를 결정할 수 있겠다.
    4. 소통 방법:
      1. 카톡 (memory) : 일상적인 소통
        1. 웹사이트 (storage)
          • 각 세션 => 아이템 (갯수 제한/ 매진/ 대기자 명단)
          • 세션 진행자 선정 (챕터별 신청을 받음/ 다수도 가능)
          • 블로그 형식 (아카이빙 => 블로그 포스팅/ 참여자 작업증명 => 댓글/ 답변 형식)
          • 이벤트 게시판
          • 투표
          • 캘린더
          • 회원가입
    5. 장소:
      1. 1) 논스
      2. 2) 블록체인 포럼 사무실 사용
      3. 3) 위워크 회의실 (역삼, 서울역)
    6. 시간:
      • 세션 시간: 고정 요일, 시간
      • 2시간
    7. 운영 인원
      • 역할
      • 웹사이트 관리
      • 세션 진행자
        • 발제 (질문)
        • 사전 준비 (참고문헌)
    8. 세션 진행 방식
      • 세션 진행자의 재량에 따라 유동적인 형태로 진행
      • 서론
      • 발제
      • 토의
      • 결론
      • 쉬는시간
    9. 작업증명
      1. 세션 진행자
      2. 세션 참여자 (댓글)
      3. 인센티브
        • (지속적인 논의/ pending question)
    10. 프로토콜 합의의 주체
      1. 참여자: 결정권을 가짐
      2. 개정권 (amendment) (지속적인 논의/ pending question)
